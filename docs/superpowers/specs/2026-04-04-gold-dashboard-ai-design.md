# 黄金分析仪表盘设计方案

**日期**：2026-04-04
**项目**：gold-dashboard 容器化 + AI 新闻分析

---

## 背景

用户持有纸黄金（中长线），希望通过新闻建立对金价走势的判断。需要：
1. 实时金价
2. 专业分析（而非原始新闻）
3. 突发风险预警

---

## 需求确认

| 功能 | 描述 |
|------|------|
| 实时金价 | OKX XAUT，24/7，单位：$/oz + ¥/g |
| 每日简报 | 每小时汇总当日新闻，MiniMax AI 生成综合判断 |
| 实时风险预警 | 每 15 分钟检测重大事件，MiniMax 判断对金价影响 |
| 输出方式 | RSS 订阅 + 网页界面 |
| 配置管理 | 信息源写入配置文件（sources.yaml），增删无需改代码 |
| 端口 | 18888 |

---

## 技术方案

### 架构

```
┌─────────────────────────────────────────────────────────┐
│                    群晖 Docker 容器                      │
│                                                         │
│  ┌──────────────┐    ┌──────────────┐    ┌───────────┐ │
│  │  RSSHub      │───▶│  news_ai.py  │───▶│ MiniMax   │ │
│  │  (已有)      │    │  定时任务     │    │ API       │ │
│  └──────────────┘    └──────────────┘    └───────────┘ │
│         │                   │                   │        │
│         ▼                   ▼                   ▼        │
│  ┌──────────────────────────────────────────────────┐   │
│  │              gold_rss_server.py                    │   │
│  │         端口 18888，金价 + 分析结果 HTTP          │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 数据流

1. 每 15 分钟，`news_ai.py` 从 RSSHub 拉取配置中指定的 RSS 源
2. 重大事件检测 → MiniMax API 判断影响程度 → 风险预警结果
3. 每小时汇总 → MiniMax 生成「今日金价简报」
4. `gold_rss_server.py` 提供：网页界面 + RSS 输出

### 信息源配置（sources.yaml）

```yaml
sources:
  - name: 华尔街见闻快讯
    url: http://192.168.2.200:11200/wallstreetcn/live
    enabled: true
  - name: 金十数据
    url: http://192.168.2.200:11200/jinse/lives
    enabled: false
```

增删信息源：修改 `sources.yaml` → 重启容器。

### MiniMax 分析策略

**风险预警**（每 15 分钟）：
- 检测关键词：关税、制裁、战争、FOMC、CPI、非农、央行、黄金、美元
- MiniMax prompt：判断该事件对金价的影响方向（利多/利空/中性）及程度（轻微/中等/重大）

**每日简报**（每小时）：
- 汇总最近 N 条相关新闻摘要
- MiniMax prompt：综合判断今日金价走势，给出操作参考

### 输出

**网页**（http://192.168.2.200:18888/）：
- 顶部：实时金价（$/oz + ¥/g）
- 中部：今日简报 + 当前风险评级
- 底部：最近新闻列表，每条标注利多/利空/中性

**RSS**（http://192.168.2.200:18888/rss）：
- 每次分析结果作为一条 RSS item
- 重大预警优先显示

### 依赖

- Python 3.14
- httpx（RSS 抓取）
- MiniMax SDK / 直接 HTTP 调用
- APScheduler（定时任务）
- FastAPI / 原生 http.server（HTTP 服务）

**不包含**：Playwright、Ollama、Chromium（轻量级）

## 容器化

```
gold-dashboard/
├── Dockerfile
├── docker-compose.yml
├── gold_rss_server.py   # HTTP 服务（金价 + 网页 + RSS）
├── news_ai.py           # 后台任务（新闻抓取 + AI 分析）
├── sources.yaml         # 信息源配置
├── frontend/
│   ├── index.html
│   ├── css/style.css
│   └── js/
└── requirements.txt
```

- 单个 Dockerfile，基于 `python:3.14-slim`
- 容器暴露端口 18888
- 数据卷挂载 `sources.yaml` 便于修改

---

## 成本估算

MiniMax API 按 token 计费：
- 每日简报：~500 token
- 风险预警（每 15 分钟 × 24 = 96 次）：~50 token/次 × 96 = 4800 token
- **合计约 5000 token/天**，MiniMax 免费额度足够个人使用。
